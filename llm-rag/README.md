# LLM RAG 方案验证

本项目致力于验证多种 RAG 数据清洗与拆分策略，评估其在复杂文档（如年报、财务报告）及跨行业场景下的检索质量表现。

## 核心目标
针对专业领域的长文档，通过对比式实验探索最优 RAG 路径。目标是在 **ERC2 比赛数据集上达到 80% 左右** 的准确度水平。

> [!NOTE]
> 本部分实验内容基于日常实践经验及前沿论文策略进行设计与验证。

---

## 正在验证的 RAG 方案

目前项目中主要对比以下三种核心路径：

### 1. 基础 HardChunk 方案 (Baseline)
- **定义**：最原始的数据物理分块策略。
- **策略**：仅根据物理字符位置进行“硬切”，不进行语义分析或元数据注入。
- **目的**：作为实验对照组，直观展示基础 RAG 在处理专业术语、实体指代以及上下文缺失时的局限性。

### 2. RAGFlow 深度布局分析方案
- **特点**：利用专业 RAG 中间件的深度文档解析能力（如 DeepDoc 模型）。
- **重点**：验证其对复杂 PDF 布局、嵌套表格以及段落层级关系的还原效果，提升检索时的语义一致性。

### 3. Markdown 层级化拆分方案
- **路径**：`PDF -> Markdown -> Hierarchical Chunking`
- **技术栈**：
    - 拟采用 **Duckling**、**Rockflow**、**PaddleOCR** 等技术手段实现高质量的文档结构还原。
- **核心逻辑**：
    1. **结构化还原**：先将非结构化 PDF 转换为标准的 Markdown 格式，保留标题层级。
    2. **层级化切片**：利用 Markdown 的标题（H1-H4）作为逻辑分界线进行拆分。
    3. **父子检索 (Parent-Child Retrieval)**：使用细粒度的小分块进行精确索引，检索时回溯其所属的大分块（Parent）以补充完整的上下文。
