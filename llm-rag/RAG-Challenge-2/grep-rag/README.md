# Grep RAG — CC，Codex等 Coding agent采用的 RAG方式，比较符合人类的直觉

## 工作原理

1. 文档直接转 Markdown 模式，LLM生成Grep命令/固定命令传关键字直接找到所需要的字段，返回给模型/Agent完事。


**无 embedding、无向量库、无语义检索。**

## 适用场景

### 一般场景

- **代码/配置/日志搜索** — 精确符号名、变量名、错误码，字面匹配就够了
- **快速原型验证**
- **规则/条款/标准文档查询** 
- **单次/临时检索任务** — 不值得为一次性查询建索引
- **文档少** — 几百文档等场景做trade off
- **模型足够强** — 用长上下文 + 强模型，模型自己能从碎片化匹配中拼出答案

### 不适合

- **模糊语义查询** — 
- **大规模知识库** — 文档量大了之后，grep 结果返回复杂
- **跨语言检索** — 中文提问搜英文文档
- **需要理解表格/图表内容的查询** — grep 检索表格不行吧应该

## 核心逻辑

> 检索越弱，模型就得越强。

本质是把检索质量的压力转移给模型的理解能力。如果模型够强，这套性价比就很好，反之就不好，额外一点，非常烧token和上下文，感觉只适合coding agent。
